# Contributing

## Overview

This is a personal GitHub profile repository maintained solely by its owner, [Alan Szmyt](https://github.com/szmyty).

It is not a collaborative open-source project. The content, structure, and automation here reflect intentional personal branding and engineering decisions made by a single author.

---

## Contribution Policy

External contributions are not accepted for this repository.

- Pull requests will not be reviewed or merged.
- Issues opened for feature requests or code changes will be closed.
- The repository is curated for portfolio presentation and automated content generation.

This is not a reflection on the quality of outside contributions — it simply reflects the nature of a personal profile system where consistency and authorship are intentional.

---

## How the Repository Works

Most content in this repository is **automatically generated** by GitHub Actions workflows:

- **Metrics** — GitHub statistics and activity graphs are generated by [lowlighter/metrics](https://github.com/lowlighter/metrics) and committed as SVG files.
- **Activity** — Recent activity in the `README.md` is updated automatically via a scheduled workflow.
- **Profile Summary Cards** — Summary card images are generated and pushed by a dedicated workflow.

Generated files should not be edited manually. Any manual changes to these artifacts will be overwritten the next time the relevant workflow runs.

---

## Development Notes

For the owner's reference:

- Changes should be intentional and minimal.
- Avoid modifying generated artifacts directly — update the workflow configuration instead.
- Workflows are the source of truth for any content they produce.
- Prefer automation over manual updates wherever possible.

---

## Learning and Reference Use

You are welcome to explore this repository for inspiration or reference.

If you find something useful, consider **forking the repository** and adapting it to your own needs rather than opening a pull request here.

The workflows, configuration, and structure are designed to be reasonably understandable and reusable.
